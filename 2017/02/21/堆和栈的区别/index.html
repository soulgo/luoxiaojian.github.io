堆和栈的区别?

管理方式?

  • 栈：编译器自动管理，无需我们手动控制；
  • 堆：需要程序员手动控制，容易产生memory leak.

申请内存大小?

  • 栈：
    • 向低内存地址扩展，是一块连续的内存区域；
    • 栈顶的地址和栈的最大容量都是系统规定的，从栈获得的内存空间比较小；
    • 如果申请的空间超过栈的剩余空间，提示overflow。
  • 堆：
    • 向高内存地址扩展，不连续的内存区域；
    • 堆的大小受限于计算机系统中的有效虚拟内存；
    • 从堆获得空间比较自由，也比较大；
    • 堆系统是由链表来存储空闲的内存地址，自然是不连续的，而且链表的遍历方向是由低地址向高地址。

分配效率

  • 栈：
    • 效率比较快；
    • 计算机在底层对栈提供支持，分配专门的寄存器存放栈的地址，压栈出栈都有专门指令执行，因此栈的效率比较高。
  • 堆：
    • 效率比较低；
    • 堆是C/C++函数库提供的，机制比较复杂，因此比较慢。

分配方式

  • 栈：栈有两种分配方式：动态和静态
    • 静态：静态分配由编译器完成，比如局部变量的分配；动态：动态分配由alloc函数进行分配，但栈的动态分配和对的动态分配不一样，栈的动态分配是由编译器进行释放，无需手动实现。
    • 堆：堆的分配方式是动态分配，没有静态分配。

碎片问题

  • 栈：栈是先进后出的队列，压栈出栈都有严格的规则，所以也就不会产生内存碎片问题。
  • 堆：对于堆来讲，频繁的new/delete会造成内存空间的不连续，从而形成大量的随便，致使效率降低。
